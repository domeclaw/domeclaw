# แผนการเพิ่มการรองรับคำสั่งภาษามนุษย์ใน dc-hotwallet SKILL.md

## เหตุผล (Why)
เพื่อให้ผู้ใช้สั่งงานกระบวนการกระจายเงิน (transfer) และการตรวจสอบยอดเงินผ่านคำสั่งภาษามนุษย์ (เช่น โอน 0.1 CLAW ให้ 0xA3570FCDA303F55e0978be450f87F885d80a3758) แทนการใช้คำสั่ง CLI ที่ซับซ้อน (/wallet transfer ...) โดยผลลัพธ์ต้องเหมือนกัน

## สิ่งที่จะแก้ไข (What Changes)
1. เพิ่มแผนผังคำสั่งภาษามนุษย์สำหรับกระบวนการกระจายเงิน
2. แมปคำสั่งภาษามนุษย์ไปยังเครื่องมือที่มีอยู่ (wallet_auto_transfer)
3. เพิ่มตัวอย่างคำสั่งภาษามนุษย์ในภาษาไทย
4. อัปเดตข้อมูลเมตาดาต้าของ skill เพื่อระบุการรองรับคำสั่งภาษามนุษย์
5. เพิ่มกฎการแยกประโยคคำสั่งภาษามนุษย์เป็นพารามิเตอร์เครื่องมือ

## ขั้นตอนการนำไปใช้ (Implementation Steps)
### ขั้นตอน 1: วิเคราะห์โครงสร้าง SKILL.md ปัจจุบัน
- อ่านไฟล์ workspace/skills/dc-hotwallet/SKILL.md เพื่อหาส่วนสำหรับเพิ่มคำสั่งภาษามนุษย์
- ตรวจสอบส่วน metadata ที่มีคีย์ nanobot.requires.tools

### ขั้นตอน 2: เพิ่มแผนผังคำสั่งภาษามนุษย์
- เพิ่มส่วน "Natural Language Commands" ใน SKILL.md
- กำหนดรูปแบบคำสั่ง: [การกระทำ] [จำนวน] [สกิล] ให้ [ที่อยู่]
- ตัวอย่างรูปแบบ: "โอน {amount} {token} ให้ {address}"

### ขั้นตอน 3: แมปคำสั่งไปยังเครื่องมือ
- เพิ่มการแมปในส่วน metadata.nanobot.natural_language
- แมปรูปแบบ "โอน {amount} {token} ให้ {address}" ไปยังเครื่องมือ wallet_auto_transfer
- กำหนดการแปลงพารามิเตอร์: amount → amount, token → token, address → to

### ขั้นตอน 4: เพิ่มตัวอย่างคำสั่งภาษามนุษย์
- เพิ่มตารางตัวอย่างคำสั่งภาษามนุษย์และผลลัพธ์ที่คาดหวัง
- ตัวอย่าง: "โอน 0.1 CLAW ให้ 0xA3570FCDA303F55e0978be450f87F885d80a3758" → เรียก wallet_auto_transfer ด้วยพารามิเตอร์ to=0xA3570FCDA303F55e0978be450f87F885d80a3758, amount=0.1, token=CLAW

### ขั้นตอน 5: อัปเดตเมตาดาต้า skill
- เพิ่มคีย์ natural_language_support: true ใน metadata.nanobot
- เพิ่มคีย์ natural_language_patterns เพื่อเก็บรูปแบบคำสั่ง

### ขั้นตอน 6: เพิ่มกฎการแยกประโยค
- เพิ่มกฎการแยกประโยคคำสั่งภาษามนุษย์เป็นพารามิเตอร์เครื่องมือ
- กฎ: แยกจำนวนจากคำสั่ง, แยกที่อยู่กระจายเงิน, แยกสกิล

## ตัวอย่างคำสั่งภาษามนุษย์ (Examples)
| คำสั่งภาษามนุษย์ | คำสั่ง CLI ที่เทียบเท่า | พารามิเตอร์เครื่องมือ |
|------------------|-----------------------|----------------------|
| โอน 0.1 CLAW ให้ 0xA3570FCDA303F55e0978be450f87F885d80a3758 | /wallet transfer 0xA3570FCDA303F55e0978be450f87F885d80a3758 0.1 1234 | to=0xA3570FCDA303F55e0978be450f87F885d80a3758, amount=0.1, token=CLAW |
| ตรวจสอบยอด CLAW ของ 0xA3570FCDA303F55e0978be450f87F885d80a3758 | /wallet balance 0xA3570FCDA303F55e0978be450f87F885d80a3758 | address=0xA3570FCDA303F55e0978be450f87F885d80a3758, token=CLAW |

## การตรวจสอบ (Verification)
1. เริ่มต้น gateway และตรวจสอบว่า skill dc-hotwallet โหลดสำเร็จ
2. ส่งคำสั่งภาษามนุษย์ผ่าน Telegram
3. ตรวจสอบว่า Picoclaw เรียกเครื่องมือ wallet_auto_transfer ด้วยพารามิเตอร์ที่ถูกต้อง
4. ตรวจสอบว่ากระบวนการกระจายเงินสำเร็จและส่ง transaction hash กลับ
